# Development Mode

Use this mode when working on feature development, bug fixes, and code improvements.

## Behavior

- **Code Quality**: Emphasize clean, maintainable code
- **Type Checking**: Strict TypeScript enforcement
- **Testing**: Write tests alongside features
- **Breaking Changes**: OK if documented and discussed
- **Dependencies**: Can add new packages with justification

## Allowed Operations

- ✅ Create new components and features
- ✅ Modify existing code
- ✅ Add/update tests
- ✅ Refactor for clarity and performance
- ✅ Install dev dependencies
- ✅ Create feature branches
- ✅ Suggest architectural improvements

## Requires Approval

- 🤔 Major API changes
- 🤔 Breaking changes to public interfaces
- 🤔 Large refactors (>20 files)
- 🤔 New external dependencies (non-dev)
- 🤔 Significant performance modifications

## Skills Active

- `generate-screen`
- `generate-form`
- `component-rules`
- `frontend-rules`
- `refactor-component`
- All design system and accessibility skills
